# 项目现状（截至 2026-03-14）

## 一、业务现状

### 1) 当前产品定位已基本收敛

项目当前已从“单点 AI 问答/内容生成”转向“AI 学习流程导航系统”主线，核心叙事围绕：

`学习目标 -> 诊断/方案 -> 路径规划 -> 分阶段学习任务 -> 训练作答 -> 反馈与下一步动作 -> 成长沉淀`

这条闭环在业务语义上已经清晰，且前后端均有对应承接。

### 2) MVP 主业务链路可运行

目前可以完成的核心业务流程包括：

- 账号登录与身份保持（进入受保护学习流程）
- 创建学习会话、查看当前会话、恢复历史会话
- 生成学习路径与分阶段任务（结构/理解/训练/反思）
- 执行任务、进入训练与测验、提交答案
- 生成学习反馈、薄弱点、会话报告与成长看板
- 新增能力诊断与学习计划（预览/确认）链路，补齐“先诊断再规划”的入口能力

从业务成熟度看，系统已具备可演示、可留痕、可复盘的 MVP 主闭环。

### 3) 当前业务优势

- **闭环完整性增强**：不仅有任务执行，还能沉淀反馈、报告和成长数据。
- **可解释性增强**：学习反馈、弱点诊断、下一步动作等能力已形成明确输出面。
- **入口能力前移**：诊断与学习计划能力已接入，减少“直接做题但缺少起点分析”的问题。

### 4) 当前业务短板（需要下一阶段收敛）

- **闭环可感知度仍可提升**：用户对“本次训练如何具体改变下一步路径”的感知仍需强化。
- **语义一致性仍有历史包袱**：阶段命名与接口兼容路径仍保留演进痕迹（存在多套历史别名）。
- **能力较分散**：模块已较多，但需要进一步在产品层形成更强的一条“主故事线”。

## 二、技术现状

### 1) 技术栈与工程形态

- **后端**：Spring Boot 3.4.3 + Java 17 + Spring JDBC + Flyway + PostgreSQL
- **前端**：Vue 3 + Vue Router + Pinia + TypeScript + Vite
- **仓库结构**：前后端分目录管理，根目录提供一键并行启动脚本（`npm run dev`）
- **工程能力**：已配置 GitHub Actions（后端 CI/CD workflow），具备基础持续交付能力

### 2) 后端能力现状

后端已形成分层结构（`api / application / domain / infrastructure`），核心接口覆盖：

- 会话与路径：创建会话、规划任务、会话总览、路径、当前会话
- 任务执行：任务详情、运行、提交
- 训练与测验：题目生成、测验状态、提交与反馈
- 学习洞察：学习报告、会话报告、成长看板、薄弱点
- 诊断与学习计划：诊断会话创建/提交、学习计划预览/确认/查询
- Tutor：消息查询、提交、流式接口

同时保留了较强兼容性（如 `/api/session` 与 `/api/sessions` 并行支持），利于平滑演进但也带来一定契约复杂度。

### 3) 数据层与模型现状

- Flyway 迁移已推进到 `V21`，从基础会话/任务模型扩展到：
  - 练习与测验模型
  - 反馈与掌握度模型
  - 诊断与能力画像
  - 学习计划与状态统一
- 数据建模已经从“单链路”升级为“会话、训练、反馈、规划”协同模型，支撑闭环追踪。

### 4) 前端能力现状

前端路由已覆盖登录、首页、诊断、学习计划、会话、任务运行、测验、报告、成长看板等关键页面，说明前端已具备完整流程承接能力。

现状特征：

- 已形成业务页面 + 业务组件 + 状态管理 + API service 的基本分工。
- 具备主链路可视化与交互承载能力，不再是纯静态原型。
- 仍处于“持续收敛”阶段，部分页面在叙事一致性和信息层次上有继续优化空间。

### 5) 测试与质量现状

- 后端已有较完整单测/集成测试分布（服务层、控制器、LLM 网关、可观测性、策略等）。
- 可见测试重点已从“能跑”转向“策略正确性、接口行为和稳定性”。
- 前端自动化测试资产目前较少（未见成体系 `spec/test` 文件），前端质量保障仍更多依赖联调与人工验证。

### 6) 当前技术风险与约束

- **接口与命名兼容成本**：为兼容历史路径保留了别名，长期需逐步收敛契约。
- **前后端一致性成本**：模块扩展较快，文档、接口、页面语义同步成本偏高。
- **AI 输出治理持续性要求高**：模型调用、fallback、可观测性已建设，但需持续保持规则兜底与可解释性。

---

## 三、一句话结论

截至目前，项目已从“基础流程原型”进入“可运行的 MVP 闭环阶段”，下一步重点不再是大幅扩功能，而是围绕统一语义、强化闭环可感知性、降低演进复杂度做收敛与打磨。
