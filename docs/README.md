# Docs Index

`docs/` is organized by usage so later documents can be added without guessing where they belong.

## Recommended layout

- `architecture/`: stable system architecture and core design
- `backend/`: backend implementation notes, flow docs, and delivery summaries
- `frontend/`: frontend implementation notes and UI-related summaries
- `devops/`: local development, deployment, and CI/CD docs
- `product-planning/`: requirement analysis, MVP scope, API contracts, and refactor proposals
- `implementation-plans/`: execution plans or design drafts for a concrete implementation task
- `prompts/`: prompt assets, prompt inventories, and LLM usage guides
- `issues/`: temporary problem notes, issue records, and pending cleanup items
- `archive/`: historical documents that should be kept but are not part of the current working set

## Naming suggestions

- Prefer kebab-case English file names for new Markdown files
- Use `YYYY-MM-DD-` prefixes for dated plans, issue records, or snapshots
- Put long-lived reference docs in topical folders; put outdated or one-off delivery notes in `archive/`
