# 领域模型收敛方案

## 目标模型（MVP）
- `learning_session`
- `learning_task`（当前代码对应 `task`）
- `learning_step`（语义层等同 `task + stage`，先不改表）
- `learning_summary`（聚合视图，先不单独建表）

## 1) learning_session（对应 `LearningSession` + `learning_session`）

### 保留字段
- `id`
- `user_pk`
- `course_id`
- `chapter_id`
- `goal_text`
- `current_node_id`
- `current_stage`
- `status`
- `last_active_at`
- `created_at` / `updated_at`

### 删除/弱化字段
- 无需新增“诊断结果”相关字段
- `user_id`（字符串冗余）后续可仅内部兼容，逐步废弃

### 预留但不启用
- `completed_at`（仅会话真正完成时写入）

## 2) learning_task（对应 `Task` + `task`）

### 保留字段
- `id`
- `session_id`
- `node_id`
- `stage`
- `objective`
- `status`
- `output_json`
- `created_at` / `updated_at`

### 删除/弱化字段
- `input_json`、`expected_output_schema`（若未被真实消费，逐步废弃）
- 对外不暴露“计划来源/风险包装字段”

### 预留但不启用
- 无新增字段，先用 `task_attempt` 记录过程证据

## 3) learning_step（语义模型）

### 当前实现映射
- `learning_step = task(stage)`

### 期望对外字段（接口层）
- `step_id`(`task_id`)
- `title`（由 stage + nodeName 生成）
- `objective`
- `stage_type`(`STRUCTURE/UNDERSTANDING/TRAINING/REFLECTION`)
- `content_summary`（来自 task output）
- `expected_output`（阶段预期产出描述）
- `status`

### 删除/弱化
- 去掉与“诊断/路线选择”绑定的 step 元数据

## 4) learning_summary（聚合输出，不建表）

### 数据来源
- `task` + `task_attempt` + `node_mastery` + `learning_session`

### 保留字段（输出层）
- `completed_steps_count`
- `total_steps_count`
- `covered_nodes`
- `next_task`
- `feedback_text`（中性、事实化）

### 删除字段
- `goal_score`、`strengths/risks` 这类无证据早期评估字段
- 固定模板总结字段

### 预留但不启用
- `confidence_level`
- `diagnostic_conclusion`
- `dynamic_plan_adjustment_reason`

## 模型收敛原则
1. 过程先于结论：没有证据就不输出能力判断
2. 状态先于包装：字段必须参与状态推进或反馈生成
3. 兼容先于重写：优先复用现有 `session/task/stage/task_attempt/node_mastery`
