# 接口收敛方案

## 保留接口（MVP）

### 1) 创建学习会话
- 路径：`POST /api/session/create`
- 职责：创建 `learning_session`，落初始上下文（course/chapter/goal）
- 输入：`course_id`、`chapter_id`、`goal_text`
- 输出：`session_id`

### 2) 生成/获取初始学习计划
- 路径：`POST /api/session/{sessionId}/plan`
- 职责：按 session 生成 3~5 个学习步骤（任务）
- 输入：`session_id`（path）
- 输出（收敛后）：
  - `session_id`
  - `tasks[]`: `task_id`, `stage`, `node_id`, `objective`, `status`
- 说明：`plan_source/plan_reasoning_summary/risk_flags` 从 MVP 响应移除

### 3) 获取当前学习计划/会话状态
- 路径：`GET /api/session/{sessionId}/overview`
- 职责：返回 timeline、当前推荐步骤、进度
- 输入：`session_id`
- 输出：`timeline[]`、`next_task`、`progress`（保留 course/chapter/goal）

### 4) 获取当前步骤内容
- 路径：
  - `GET /api/task/{taskId}`
  - `POST /api/task/{taskId}/run`
- 职责：读取步骤详情并生成该步骤学习内容
- 输入：`task_id`
- 输出：步骤详情 / 运行结果（包含 `output`）

### 5) 完成当前步骤
- 路径：`POST /api/task/{taskId}/submit`（训练步骤）
- 职责：提交训练答案，更新证据、mastery、next_task
- 输入：`user_answer`
- 输出：`score/feedback/next_task`
- 说明：非 TRAINING 步骤以 `run` 成功即视为完成

### 6) 获取学习总结（阶段反馈）
- 路径：
  - 短期复用：`GET /api/session/{sessionId}/learning-feedback/weak-points`（改名语义）
  - 建议新增：`GET /api/session/{sessionId}/summary`
- 职责：基于真实过程数据给出事实反馈
- 输入：`session_id`
- 输出建议：
  - `completed_steps`
  - `covered_nodes`
  - `current_progress`
  - `next_recommended_step`
  - `process_feedback_text`（中性措辞）

## 旧接口删除/废弃
1. 删除：`POST /api/session/goal-diagnose`
- 理由：无证据评估

2. 删除：`POST /api/session/path-options`
- 理由：固定路线与计划生成无因果

3. 删除：`GET /api/session/{sessionId}/plan-preview`
- 理由：MVP 非主链路字段包装

4. 弱化：`GET /api/session/{sessionId}/path`
- 理由：可作为调试接口，不进入主流程

## 前端接口映射调整
1. 移除 `frontend/src/api/session.ts`：`diagnoseGoal/getPathOptions`
2. `frontend/src/stores/session.ts` 移除 `goalDiagnosis/pathOptions/selectedPathId`
3. `frontend/src/views/SessionView.vue` 不再请求诊断与路线接口，仅依赖 `overview + task run/submit + summary`
