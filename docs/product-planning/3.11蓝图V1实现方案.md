# 二、开发路线图（双人协作版）

你们只有两个人，所以路线图必须遵循三个原则：

1. **先收口主线，不要平行开太多新战场**
2. **前后端每阶段都能独立产出可见成果**
3. **优先做比赛最能看见的东西：闭环、图谱、报告、展示感**

我建议分成 **4 个阶段**。

---

## Phase 0：统一基线（1~2 天）

这是必须做的，不然越做越乱。

### 目标

统一术语、统一主故事、统一页面路径、统一对象命名。

### 后端同学

* 统一阶段术语，对外统一成一套表达，至少前后端返回一致
  建议对外统一为：

  * 建立框架
  * 理解原理
  * 训练应用
  * 评估反思
* 梳理当前 API 与状态流：

  * session
  * task
  * practice
  * feedback
  * tutor
* 输出一份当前接口清单和字段定义

### 前端同学

* 统一页面命名、路由命名、文案风格
* 把现有 Session / TaskRun / TaskSubmit / History 的信息结构画成页面草图
* 整理组件复用边界：卡片、状态区、报告区、图谱区、Tutor面板

### 阶段产出

* 一份统一术语表
* 一份主流程图
* 一份接口和页面映射表

---

## Phase 1：做实“训练驱动闭环”（优先级最高，3~5 天）

这是比赛里最核心的一段，因为你们已经有能力骨架了，应该先把它做成“用户明显能感知到”。

### 目标

让用户明确看到：

**训练结果 -> 为什么这样判断 -> 下一步为什么这么安排**

### 后端同学

重点做三件事：

#### 1. 聚合学习反馈报告接口

新增或完善一个聚合接口，返回：

* 本轮总体表现
* 每题评价
* 薄弱点
* 当前掌握结论
* 推荐下一步
* 推荐原因

#### 2. 统一训练事实源

以 `practice_item + practice_submission + node_mastery + learning_feedback` 为主线，避免旧提交逻辑继续分叉。
因为你们现状里已经有训练、反馈、掌握度能力，下一步应该把它们聚合成单一故事。

#### 3. 明确下一步决策规则

先别追求太智能，用规则就够：

* 分数高 + 错误少 → 推进下一知识点
* 部分掌握 → 插入复习任务
* 掌握差 → 回退前置点 / 继续强化

### 前端同学

重点做三件事：

#### 1. 重构学习报告页

报告页必须从“结果页”变成“导航页”：

* 学会了什么
* 哪些地方薄弱
* 当前节点状态
* 系统建议下一步
* 为什么这么建议

#### 2. Session 页增加闭环摘要

增加：

* 最近一次训练结果
* 当前薄弱点
* 路径调整说明
* 推荐继续按钮

#### 3. 训练提交后的回流体验

用户做完题后，不要只是结束，要有：

* 评估中
* 结果已生成
* 推荐动作入口

### 阶段产出

* 可演示的“学习—检测—反馈—下一步”闭环
* 一个像样的报告页
* 一个更强的 Session 中枢页

---

## Phase 2：补“私人知识图谱 + 成长视图”（比赛观感提升最大，4~6 天）

这是你们项目最容易“眼前一亮”的一层，也是老师提到未来更灵活个性化课程的最好支撑。

### 目标

让系统从“单次学习工具”升级成“长期成长系统”。

### 后端同学

#### 1. 增加用户知识图谱聚合查询

不一定先做复杂图数据库，基于现有 `concept_node + node_mastery` 就能做第一版。
返回：

* 节点列表
* 当前掌握度
* 节点状态（未学 / 学习中 / 待巩固 / 已掌握）
* 关联关系
* 当前推荐路径

#### 2. 每轮学习完成后更新用户节点状态

规则可先简单：

* 首次学习完成 → 新增用户节点记录
* 训练后更新 mastery_score
* 生成弱项标签和最后学习时间

#### 3. 成长统计接口

输出：

* 已学节点数
* 当前课程总体掌握度
* 最近 7 次训练表现
* Top 弱项节点

### 前端同学

#### 1. 做“私人知识图谱页”

第一版不用炫技，做成：

* 节点网格 / 关系图
* 颜色代表掌握度
* 当前节点高亮
* 推荐路径高亮

#### 2. 做“成长仪表盘”

包括：

* 已学习节点数
* 课程完成进度
* 训练趋势
* 当前薄弱领域

#### 3. 把知识图谱更新嵌入报告页

学习完一个节点后提示：

* 已加入私人知识图谱
* 当前掌握度变化
* 后续建议路线

### 阶段产出

* 个人知识图谱
* 学习成长仪表盘
* 长期个性化的产品感

---

## Phase 3：补“能力分析 + 个性化方案页”（系统高度提升，3~5 天）

这是让评委觉得“这不是普通学习平台”的关键。

### 目标

在学习前就体现“个性化”。

### 后端同学

#### 1. 用户能力分析接口

* 当前会话任务方案预览

### 前端同学

#### 1. 能力分析页

做成“AI诊断你现在状态”的感觉：

* 当前水平
* 风险提示
* 重点知识
* 推荐节奏

#### 2. 方案确认页

展示：

* 本轮学习将学什么
* 为什么这样安排
* 预计需要几个阶段
* 确认开始学习

### 阶段产出

* 学前分析页
* 方案确认页
* 个性化感更强的起点体验

## Phase 4：打磨竞赛展示版（2~4 天）

这一步不是新增大量能力，而是让你们真的能拿去比赛讲。
