# 二、本轮前端目标

本轮只聚焦 4 件事，按顺序做：

## 目标 1：把 Session 页做成真正的中枢页

当前 Session 页不应只是步骤展示，而应承担“学习导航中枢”的职责。

必须让用户在 Session 页上明确看到：

- 当前学习主题/知识点
- 当前位于哪一阶段
- 最近一次训练结果摘要
- 当前薄弱点
- 系统建议的下一步动作
- 为什么这样建议

### 实现要求

1. 梳理 Session 页信息层级，避免信息墙。
2. 页面结构优先考虑：
   - 顶部：当前学习概览
   - 中部：当前任务序列 / 当前阶段
   - 侧边或下方：最近训练摘要 + 当前薄弱点 + 推荐下一步
3. 如果后端当前没有完整聚合接口，先兼容现有数据来源，但在代码中保留清晰 TODO，标明未来应改为聚合接口驱动。
4. 不允许暴露内部混乱状态名给用户。

---

## 目标 2：重构 TaskRun 页，让它聚焦“当前行动”

TaskRun 页必须从“内容堆砌页”收敛为“主学习区 + 辅助区”。

### 页面结构要求

主区域：
- 当前任务标题
- 当前任务目标
- 当前学习内容
- 明确的主要操作按钮（继续学习 / 进入检测 / 提交本阶段）

辅助区域：
- 当前阶段说明
- Tutor 助手入口/弹出面板
- 当前学习提示
- 必要时显示推荐关注点

### 实现要求

1. 页面主次必须清楚，用户第一眼看到的是“现在该学什么”。
2. Tutor 必须更像跟随式助手，而不是喧宾夺主的聊天页。
3. 减少重复文案、重复状态块、重复按钮。
4. 将阶段说明产品化，不要直接输出裸枚举值。

---

## 目标 3：把 TaskSubmit / 学习报告页做成“结果 + 导航”页面

用户提交检测后，系统不能只显示结果，而要清楚回答：

- 我学得怎么样
- 哪些地方有问题
- 系统怎么判断的
- 下一步应该继续、复习还是强化

### 页面结构要求

建议包含：

1. 结果总览
   - 本轮表现
   - 掌握情况结论

2. 细项反馈
   - 每题表现或关键问题摘要
   - 薄弱点/易错点

3. 下一步建议
   - 推荐动作
   - 推荐原因
   - 下一步按钮（继续当前节点 / 进入复习 / 进入下一节点）

4. 成长沉淀提示
   - 当前学习结果已计入个人成长记录/知识图谱（如果已有对应能力）

### 实现要求

1. 页面语义从“结束页”改为“导航页”。
2. 支持 `评估中 -> 结果生成 -> 推荐动作` 的状态过渡。
3. 如果已有 PracticeStatusCard、NextStepPanel 等组件，优先复用或轻量升级。

# 三、UI/UX 统一原则

你必须遵守以下设计原则：

## 1. 极简、引导式

不是把信息全塞给用户，而是让用户知道“此刻下一步做什么”。

## 2. 主次分明

每个页面只能有一个主动作。

## 3. 语义一致

同一概念只用一套说法。

## 4. 产品化表达

不要把数据库/枚举/内部状态直接展示给用户。

## 5. 可演示优先

页面要有“老师一看就懂”的讲述性。
