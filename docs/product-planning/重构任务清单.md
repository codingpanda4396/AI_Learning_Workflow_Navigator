# 重构任务清单

## P0 必做
1. 下线伪评估接口与依赖
- 后端：删除 `SessionController` 的 `/goal-diagnose`；删除 `GoalDiagnosisService` 与 `GoalDiagnosisRequest/Response`
- 前端：删除 `frontend/src/api/session.ts` 的 `diagnoseGoal`；删除 `frontend/src/stores/session.ts` 的 `goalDiagnosis/diagnosingGoal`
- 页面：删除 `frontend/src/views/SessionView.vue` 的“目标诊断”卡片

2. 下线写死路线接口与依赖
- 后端：删除 `SessionController` 的 `/path-options`；删除 `PathOptionsService` 与 `PathOptionsResponse`
- 前端：删除 `getPathOptions`、`pathOptions/selectedPathId/fetchingPathOptions`
- 页面：删除 `SessionView.vue` 的路径选项区块

3. 收敛流程步骤为 3 步
- `SessionView.vue`、`HomeView.vue`、`frontend/src/types/workflow.ts`（如存在）统一改为：计划、执行、反馈
- 去掉仅展示作用且不影响状态推进的 Step 状态计算

4. 收敛计划接口返回
- `PlanSessionResponse` 去掉 `plan_source/plan_reasoning_summary/risk_flags`
- `PlanSessionTasksService` 仅返回 `session_id + tasks`
- 前端 mapper 保持兼容最小字段

5. 阶段反馈改为“过程事实反馈”
- 复用 `WeakPointDiagnosisService` 计算逻辑，重命名为 `SessionSummaryService`
- 输出字段改为事实陈述：完成步骤、覆盖节点、建议下一步，不输出掌握度断言

## P1 应做
1. 精简计划策略暴露面
- 下线 `SessionController` 的 `/plan-preview`
- 计划模式参数 `mode` 对外隐藏，内部固定 `auto`

2. 收敛 Task 输出结构
- 在 `TaskRunnerService.generateByStage` 中移除固定“总结/下一步建议”模板语句
- 优先依赖 LLM 结果；fallback 仅保留中性结构化字段

3. 收敛 Session 概览字段
- `SessionOverviewResponse` 里 `mastery_summary` 从主页面次要化（保留但不作为“结论”展示）
- 前端反馈页只展示过程数据（完成数、阶段、下个任务）

4. 文案收敛
- 页面和接口描述中统一去掉“诊断、薄弱点、个性化路线”承诺
- 改为“目标理解、学习计划、阶段反馈”

## P2 可延后
1. 数据模型命名统一
- `Task` 在领域层等价 `learning_step`；后续可补 `LearningStep` 别名模型（不改表）

2. 新增显式 summary 接口
- `GET /api/session/{id}/summary`，聚合 `overview + node_mastery`
- 前端反馈页切换到该接口

3. 兼容期处理
- 为废弃接口增加 410/Deprecation Header，保留 1 个版本窗口

4. 测试补齐
- 删除旧接口后补控制器契约测试
- 会话主链路补集成测试：create -> plan -> run/submit -> summary
