# 最小重构实施方案

## 目标
在不大改数据库和核心执行链路前提下，把系统收敛为：创建会话 -> 生成计划 -> 分步执行 -> 过程反馈。

## 实施范围（最小成本）

### 一、后端
1. `SessionController` 收口
- 删除方法：`diagnoseGoal`、`pathOptions`、`previewPlan`
- 保留方法：`createSession`、`planSession`、`getOverview`、`getCurrentSession`

2. 下线伪能力类
- 删除类：`GoalDiagnosisService`、`PathOptionsService`
- 删除 DTO：`GoalDiagnosisRequest/Response`、`PathOptionsResponse`、`PlanPreviewResponse`

3. 收敛计划返回
- 修改 `PlanSessionResponse`：仅保留 `session_id + tasks`
- 修改 `PlanSessionTasksService.execute`：不再传递 `planSource/planReasoningSummary/riskFlags`

4. 保留并改造反馈能力
- 将 `WeakPointDiagnosisService` 改名为 `SessionSummaryService`
- 将 `LearningFeedbackController` 改名为 `SessionSummaryController`
- 输出改为“过程事实反馈”，避免“掌握/薄弱点”断言

### 二、前端
1. API 层删减
- 删除 `frontend/src/api/session.ts`：`diagnoseGoal`、`getPathOptions`

2. Store 层删减
- 删除 `frontend/src/stores/session.ts`：
  - 状态：`goalDiagnosis/pathOptions/selectedPathId`
  - action：`diagnoseGoal/fetchPathOptions`
  - loading：`diagnosingGoal/fetchingPathOptions`

3. 页面收敛
- `SessionView.vue` 四步改三步：
  - Step1 学习计划（timeline/nextTask）
  - Step2 分步学习（task run/submit）
  - Step3 阶段反馈（summary）
- 删除“目标诊断结果”和“路径选项”区块

### 三、兼容策略
1. 短期允许保留旧后端类 1 个迭代周期（标记 `@Deprecated`）
2. 前端先完成切换，再删除旧接口实现

## 建议删除类
- `backend/src/main/java/com/pandanav/learning/application/service/GoalDiagnosisService.java`
- `backend/src/main/java/com/pandanav/learning/application/service/PathOptionsService.java`

## 建议改名类
- `backend/src/main/java/com/pandanav/learning/application/service/WeakPointDiagnosisService.java` -> `SessionSummaryService`
- `backend/src/main/java/com/pandanav/learning/api/controller/LearningFeedbackController.java` -> `SessionSummaryController`

## 建议下线逻辑
- `/session/goal-diagnose`
- `/session/path-options`
- `/session/{id}/plan-preview`
- Session 页“路线选择”状态计算

## 落地后效果
1. 用户看到的每一步都能对应真实状态变化
2. 减少伪智能承诺，提升一致性与可信度
3. 后续接入真实评估和动态调度时，不需要推翻主链路

面向大学生的 AI 学习任务拆解与分步引导系统，基于学习过程提供阶段反馈。
