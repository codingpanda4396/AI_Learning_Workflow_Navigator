## 3. 主工作台怎么长

### 3.1 当前任务卡：页面绝对主角

一张大卡就够，别分裂成很多小卡片。

这张卡建议固定五段：

**第一段：当前阶段标签**

* `当前阶段：STRUCTURE`
* `当前动作：先搭起 DFS / BFS 的知识骨架`

**第二段：本步目标**

* `你现在要先区分两者的遍历推进方式，而不是先背定义。`

**第三段：为什么做这一步**

* `如果这里不区分清楚，后面会把“搜索顺序”和“使用场景”混在一起。`

**第四段：产出要求**

* `你的回答需要包含：`

  * `遍历推进逻辑`
  * `依赖的数据结构`
  * `典型适用场景`

**第五段：完成标准**

* `当你能不用例子、只靠自己的话把这三点说清，这一步就算通过。`

这张卡的价值在于，它把 prompt 从“隐形的后台字符串”变成了**用户可感知的认知任务说明**。这正好对应你项目里“Prompt 工程产品化、四阶段学习脚手架、用户输出优先”的创新叙事。

## 4. prompt 脚手架怎么体现，才会高级

这里是关键。

**不要把 prompt 直接显示成一大段长文本。**
那样会像“给用户看系统提示词”，很土，也很重。

你应该把 prompt 脚手架拆成**可执行的认知支架**，也就是把 prompt UI 化。

### 正确呈现方式：脚手架卡片

标题：
**系统给你的思考支架**

下面不是长 prompt，而是 3 个渐进式支架：

#### 支架 1：先想什么

* `先只说 DFS 和 BFS 的推进方式有什么不同`

#### 支架 2：再补什么

* `再补充它们分别依赖什么数据结构`

#### 支架 3：最后落到哪里

* `最后各写一个更典型的使用场景`

这样评委一眼就能明白：

**你不是让用户“自己写 prompt”，而是系统把高质量 prompt 拆成了学习动作。**

### 再加一个层次：提示强度分级

给每一步支架提供三级提示，不要一次全给：

* `轻提示`：一句方向提醒
* `标准提示`：给出思考框架
* `加强提示`：给出半结构模板

例如：

**轻提示**
`先不要举例，先区分“怎么走”。`

**标准提示**
`你可以从“访问顺序 / 辅助结构 / 适合的问题类型”三个角度来想。`

**加强提示**
`DFS 更像______，通常依赖______；BFS 更像______，通常依赖______。`

这就非常像“脚手架”，而不是“答案提示”。

## 5. 用户产出区要怎么设计

这里不能再是一个“问点什么”的聊天输入框。

应该是**结构化表达区**。

### STRUCTURE 阶段推荐形态

用**分段填写**，不要纯大文本。

```text
[1] DFS 的推进方式：
[2] BFS 的推进方式：
[3] DFS 常依赖的数据结构：
[4] BFS 常依赖的数据结构：
[5] DFS 更常见的场景：
[6] BFS 更常见的场景：
```

这样有三个好处：

1. 降低空白恐惧
2. 强制用户按你要的结构表达
3. 反馈可以精确回到字段，不会变成模型作文

### 输入区下方要有一句固定引导

比如：

`先写出你当前能确认的部分，不要求一次写满。`

这句话很重要，因为它降低了开始门槛。

## 6. 反馈区必须结构化，不能是大段作文

提交后，不要给一大段 AI 回复。
要给一个**教学反馈板**。

结构固定成四块：

### 你已经说对了什么

* `你已经区分出了 DFS 和 BFS 的推进方向。`

### 你漏了什么

* `还没有明确说出两者各自依赖的数据结构。`

### 你混淆了什么

* `你把“访问顺序”和“适用场景”混在一起了。`

### 下一步该怎么修

* `先补上栈/队列，再重新写场景，不要急着举复杂例子。`

这四块会让页面非常像“教学系统”。
它也和你现在任务执行/脚手架引擎强调“记录证据、判断是否推进下一状态”的后端思路是对齐的。

## 7. 一个很重要的设计点：大阶段 + 小动作双层状态

你现在其实已经有两套状态：

一套是专题脚手架四阶段：

* STRUCTURE
* UNDERSTANDING
* TRAINING
* REFLECTION

另一套是任务执行内状态机：

* ORIENT
* EXPLORE
* SELF_EXPLAIN
* CHECK
* REMEDIAL
* PASS

这两个不要打平。
应该做成**双层状态可视化**。这和你现有后端、前端架构是吻合的。 

### 页面上怎么表现

左侧纵向展示**大阶段**。
主卡标题下展示**当前小动作**。

例如：

* 当前阶段：`STRUCTURE`
* 当前动作：`SELF_EXPLAIN`
* 动作提示：`请用自己的话说清 DFS/BFS 的骨架区别`

这样页面会显得非常专业，因为它不是“只展示流程”，而是“展示当前认知动作”。

## 8. 四个阶段的页面气质必须真的不同

这是你这页能不能打动评委的关键。

### A. STRUCTURE：搭骨架

页面关键词：
**分清、归位、建立边界**

交互组件建议：

* 对比式结构输入
* 概念关系小图
* 必填字段型产出区

页面气质：

* 冷静、清晰、框架感强
* 少解释，多归纳

---

### B. UNDERSTANDING：讲机制

页面关键词：
**因果、过程、为什么这样**

交互组件建议：

* 因果链填写
* “如果没有它会怎样”追问
* 机制步骤排序

产出区不要再是“概念对比”，而应变成：

```text
[问题] 它要解决什么问题？
[机制] 它是怎么一步步工作的？
[结果] 为什么会产生这种效果？
```

页面气质：

* 更像推演板
* 强调过程箭头、步骤关系

---

### C. TRAINING：逼表达

页面关键词：
**复述、验证、暴露错误**

交互组件建议：

* 用自己的话解释
* 给一个例子验证
* 判断一个错误说法为什么错

产出区更像“说出来”：

* `请你向一个刚学图遍历的人解释 DFS 和 BFS 的差别`
* `再用一个场景说明什么时候更该先想到 BFS`

页面气质：

* 更强参与感
* 更像口头答辩

---

### D. REFLECTION：收敛规则

页面关键词：
**错因、决策规则、迁移能力**

交互组件建议：

* 错误模式提取
* 下次判断规则填写
* 一句话能力总结

结构建议：

```text
我这轮最容易错在：
因为我总会把：
所以我以后判断时先看：
下次再遇到类似题，我先检查：
```

页面气质：

* 收束、沉淀、形成方法
* 不再追求铺陈，而是结论化

## 9. AI 导师该怎么放

你的直觉是对的：**必须降为辅助位。**

### 最佳方案

默认只显示一个悬浮按钮：

`不懂这一步？`

点开后出现右侧抽屉，不是常驻大面板。

抽屉顶部只给 3 个快捷动作：

* `帮我解释这一步要求`
* `给我一个更容易理解的提示`
* `这个术语是什么意思`

不要开放式写：
`你可以问我任何问题`

因为那会把整个页面重新拉回聊天产品。

### AI 导师的产品定位

它不是老师本体。
它是：

**当前任务的辅助解释器**

## 10. 视觉层级建议

你现在最该追求的是：**一眼看懂现在该干什么。**

所以视觉上要遵守这三个原则。

### 原则 1：一个主卡压住全页

当前任务卡必须最大、最醒目。

### 原则 2：弱化说明文字

说明不要成段。
尽量改成：

* 一句目标
* 一句原因
* 三条要求

### 原则 3：所有辅助内容折叠化

这些都不该常驻展开：

* 阶段解释
* AI 长回复
* 历史消息
* 补充资料
* 延伸知识
