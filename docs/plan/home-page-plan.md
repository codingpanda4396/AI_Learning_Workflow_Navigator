# 首页改造计划

## 页面定位
- 页面对应：`frontend/src/views/GoalInputView.vue`
- 页面职责：帮助用户快速进入一轮 AI 学习流程，明确“我现在要开始什么主题”
- 设计目标：低负担、强引导、立即进入学习目标

## 当前判断
- 当前首页整体方向已经可用，主要问题不是信息缺失，而是首页还可以进一步强化“进入工作流”的确定感
- 现有内容已经具备主题选择、继续学习、底部主 CTA，适合作为轻量优化页，而不是大改信息架构

## 改造目标
- 让首屏更清楚回答“你现在在哪”和“下一步做什么”
- 弱化“选卡片浏览”的感觉，强化“开始一轮学习任务”的感觉
- 把首页从“主题目录”拉向“学习入口”

## 计划调整

### 1. 首屏结构微调
- 保留当前双区结构，但把左侧标题区改成更强的任务启动语义
- 首屏标题下只保留一句核心引导，不再铺太多说明
- 右侧“继续学习”卡保留，但视觉上降一级，避免和主启动动作抢中心

### 2. 主题选择区优化
- 保留“学科 -> 主题”两层结构
- 压缩学科卡描述，减少解释性文案
- 强化当前已选主题的视觉闭环，让用户更容易确认“这就是我要开始的内容”
- 对未开放主题继续保留占位，但进一步弱化存在感，避免干扰决策

### 3. 底部固定行动区优化
- 保留底部固定 CTA，因为它符合“单一主动作”原则
- 将底部摘要改成更明确的工作流语言，例如“即将进入诊断”
- CTA 文案应更像动作触发，而不是一般跳转按钮

### 4. 产品感强化
- 在首屏增加很轻量的流程提示：目标 -> 诊断 -> 规划 -> 执行 -> 报告
- 该提示只做定位，不做详细解释，避免抢占主内容
- 让用户在首页就知道这不是聊天产品，而是一轮学习工作流

## 实施顺序
1. 调整 `GoalInputView.vue` 的首屏文案层级与布局节奏
2. 压缩学科卡与主题卡文案长度
3. 优化底部固定栏的摘要与 CTA 文案
4. 增加轻量流程定位信息，并检查移动端首屏高度

## 验收标准
- 用户进入页面后，首屏 3 秒内能知道这是“开始学习流程”的入口
- 页面第一屏主动作只有一个，不出现多中心竞争
- 已选主题和下一步动作关系清晰
- 移动端底部固定栏不遮挡核心选择内容

## 备注
- 首页按文档要求属于“可微调优化”，不建议在这一轮引入新抽象或复杂组件
- 优先复用现有 `AppTopBar`、按钮和卡片样式体系
