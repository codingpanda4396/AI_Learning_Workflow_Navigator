# 错误处理

## 1. 统一响应格式

成功时：

```json
{
  "code": "OK",
  "message": "success",
  "data": { ... }
}
```

失败时：

```json
{
  "code": "BAD_REQUEST",
  "message": "具体错误信息",
  "data": null
}
```

---

## 2. 通用响应码（ResponseCode）

| code | 说明 |
|------|------|
| OK | 成功 |
| BAD_REQUEST | 请求参数错误（含校验失败） |
| NOT_FOUND | 资源不存在 |
| CONFLICT | 状态冲突 |
| INTERNAL_ERROR | 服务器内部错误 |

---

## 3. 业务错误码（BusinessErrorCode）

| code | HTTP 状态 | 说明 |
|------|-----------|------|
| RESOURCE_NOT_FOUND | 404 | 目标、诊断、计划、会话、任务等不存在 |
| INVALID_ARGUMENT | 400 | 参数无效（如 goalId 无效） |
| DIAGNOSIS_NOT_COMPLETED | 409 | 诊断未完成即调用后续接口 |
| PLAN_NOT_COMMITTED | 409 | 计划未确认即进入执行 |
| TASK_NOT_CURRENT | 409 | 提交完成的任务不是当前任务 |
| TASK_ALREADY_COMPLETED | 409 | 任务已完成，重复提交 |
| SESSION_NOT_COMPLETED | 409 | 会话未完成时执行不允许的操作 |
| SESSION_ALREADY_COMPLETED | 409 | 会话已完成后继续执行 |
| DIAGNOSIS_ALREADY_COMPLETED | 409 | 诊断已完成，重复提交 |
| PLAN_ALREADY_COMMITTED | 409 | 计划已确认，重复 commit |
| INVALID_REQUEST | 400 | 非法请求 |
| INTERNAL_ERROR | 500 | 内部错误 |

---

## 4. 校验失败（MethodArgumentNotValidException）

当请求体验证失败（如 `@Valid` 不通过）时：

- HTTP：400
- `code`：`BAD_REQUEST`
- `message`：`{field}: {默认错误信息}`，如 `rawGoalText: must not be blank`

---

## 5. 前端处理建议

1. **统一拦截**：在 HTTP 客户端或 axios 拦截器中根据 `response.data.code !== 'OK'` 判断失败
2. **友好文案**：可根据 `code` 映射为前端展示文案
3. **状态机校验**：调用前检查本地状态，避免触发 `DIAGNOSIS_NOT_COMPLETED`、`PLAN_NOT_COMMITTED` 等
4. **404 处理**：`RESOURCE_NOT_FOUND` 时引导用户回到目标输入或刷新

---

## 6. 典型错误场景

| 场景 | 可能 code | 建议 |
|------|-----------|------|
| 创建诊断前未创建目标 | RESOURCE_NOT_FOUND | 提示先完成目标输入 |
| 提交诊断前未创建诊断会话 | RESOURCE_NOT_FOUND | 先调用 create session |
| 预览计划前未完成诊断 | DIAGNOSIS_NOT_COMPLETED | 先完成诊断提交 |
| 获取当前任务时会话不存在 | NOT_FOUND / RESOURCE_NOT_FOUND | 检查 sessionId |
| 完成任务时 sessionId 与当前不匹配 | TASK_NOT_CURRENT | 使用最新 sessionId |
| 重复提交诊断答案 | DIAGNOSIS_ALREADY_COMPLETED | 禁用提交或提示已完成 |
