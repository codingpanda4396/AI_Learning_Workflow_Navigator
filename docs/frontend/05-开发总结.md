## 页面与接口对应

| 页面 | 接口 |
|------|------|
| 目标输入 | POST /api/goals |
| 诊断 | POST /api/diagnosis/sessions（进入时）, POST /api/diagnosis/submissions（提交） |
| 计划 | POST /api/learning-plans/preview（进入时）, POST /api/learning-plans/commit（确认） |
| 任务执行 | GET /api/sessions/{sessionId}/current-task, POST /api/tasks/{taskId}/complete |
| 报告 | GET /api/sessions/{sessionId}/report, POST /api/sessions/{sessionId}/next-action |

## 风格统一

- 所有页面使用同一套设计 tokens（色彩、圆角、间距）
- 统一组件：FormCard、PrimaryButton、SecondaryButton、PageContainer、AppTopBar、WorkflowStepper
- 统一 Loading / Error / Empty 态
- 顶栏固定 WorkflowStepper 显示当前步骤

## 比赛展示优化

- 流程步骤清晰可见（WorkflowStepper）
- 表单与展示卡片布局一致
- 错误提示友好（toast + errorCodeLabels 映射）
- 主链路可完整跑通，支持演示场景「理解链表」
