# 前端开发概述

## 1. 项目定位

**AI Learning Workflow Navigator** 是面向计算机学习场景的 AI 学习导航系统 MVP。  
不是通用 AI 问答工具，也不是自由聊天式学习产品。

**唯一主线：**

```
目标输入 → 用户诊断 → 学习规划 → 任务执行 → 反馈与 Next Action
```

## 2. 开发环境

| 项目 | 说明 |
|------|------|
| 后端地址 | `http://localhost:8080`（默认） |
| API 前缀 | `/api` |
| CORS | 已配置，`/api/**` 允许 `*` origin、`GET/POST/PUT/DELETE/OPTIONS` |

## 3. 主链路说明

| 阶段 | 用户动作 | 前端需调用 | 产出物 |
|------|----------|------------|--------|
| 1. 目标输入 | 填写学习目标表单 | `POST /api/goals` | goalId, structuredGoal, goalContextSnapshot |
| 2. 用户诊断 | 回答诊断问题 | `POST /api/diagnosis/sessions` → `POST /api/diagnosis/submissions` | diagnosisId, learnerProfileSnapshot |
| 3. 学习规划 | 预览并确认计划 | `POST /api/learning-plans/preview` → `POST /api/learning-plans/commit` | sessionId, planId, taskSequence |
| 4. 任务执行 | 执行任务、完成任务 | `GET /api/sessions/{id}/current-task`、`POST /api/tasks/{id}/complete` | taskExecutionRecord, nextTaskId |
| 5. 反馈 | 查看报告、确认下一步 | `GET /api/sessions/{id}/report`、`POST /api/sessions/{id}/next-action` | learningReport, nextActionDecision |

## 4. 前端页面建议（MVP）

| 页面 | 作用 |
|------|------|
| 目标输入页 | 收集 rawGoalText、timeBudget、selfReportedLevel 等 |
| 诊断页 | 展示诊断问题、提交 answers |
| 计划预览页 | 展示 recommendedEntry、stages、tasks、successCriteria，确认后 commit |
| 任务执行页 | 展示当前任务、推荐提问、完成标准，提交完成 |
| 学习报告页 | 展示 resultStatus、completedProgress、nextAction |

## 5. 响应格式约定

所有 API 均使用 `GlobalResponse` 格式：

```json
{
  "code": "OK",
  "message": "success",
  "data": { ... }
}
```

- 成功：`code` 为 `"OK"`，`data` 为业务数据
- 失败：`code` 为业务错误码或 `BAD_REQUEST`、`NOT_FOUND` 等，`data` 为 `null`

详见 [04-错误处理.md](./04-错误处理.md)。
